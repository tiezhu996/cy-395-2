## 项目主要需求

### 项目概述
**项目类型：纯后端服务**
搭建一个汇率换算与定时推送 API 服务，提供实时汇率查询、历史汇率、货币换算，支持用户订阅汇率变动通知。

### 功能需求

1. **实时汇率查询**：查询指定货币对的实时汇率（如 USD/CNY、EUR/CNY），支持批量查询多组货币对。
2. **历史汇率查询**：查询指定日期范围内的历史汇率数据，返回日级别汇率曲线数据。
3. **货币换算**：输入源货币、目标货币、金额，返回换算后的金额，支持链式换算（A→USD→B）。
4. **支持货币列表**：返回系统支持的全部货币代码、名称、符号列表（至少支持 30 种主要货币）。
5. **汇率订阅通知**：用户订阅特定货币对的目标汇率，当实际汇率达到目标时通过 Webhook/邮件通知（模拟发送）。
6. **定时数据更新**：后台定时任务每小时从外部汇率 API（如 exchangerate-api.com 免费接口）拉取最新汇率并存储。
7. **API Key 管理**：用户注册后生成 API Key，接口调用需携带 API Key 进行鉴权，记录调用量。
8. **调用统计**：查询指定时间段内的 API 调用次数、各接口调用分布。

### 技术要求

- 后端框架：Python + FastAPI
- ORM：SQLAlchemy
- 数据库：PostgreSQL
- 认证：API Key（Header 鉴权）
- 定时任务：APScheduler
- HTTP 客户端：httpx（调用外部汇率 API）
- 代码规范：严格遵循单一职责原则，每个文件只负责一个功能；常量（错误码、货币代码枚举、配置项）统一维护在 `backend/app/constants/` 目录下；日志统一使用 Python logging 模块，日志配置在 `backend/app/utils/logger.py`；异常信息统一定义在 `backend/app/constants/errors.py`，后端通过 FastAPI 异常处理器统一捕获并返回标准格式响应。

## 部署要求

项目必须支持 Docker Compose 一键部署。

需要提供以下文件：
- 根目录 `docker-compose.yml`（编排后端 + 数据库）
- 根目录 `.env`：包含 `COMPOSE_PROJECT_NAME=forexapi`
- `backend/Dockerfile`
- `.env.example`

docker-compose.yml 要求：
- **不要写 `version:` 字段**
- 顶层声明 `name: forexapi`
- 容器名带 `forexapi-` 前缀
- 数据库数据使用命名卷持久化
- 数据库配置 `healthcheck`
- 端口映射：
  - 后端：19410:8000
- 访问 Swagger 文档：http://localhost:19410/docs
